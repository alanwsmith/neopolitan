Welcome to Neopolitan

Introduction

Neopolitan is a plain-text file format. It's like Markdown, but with more features.

How It Works

The Basics


Neopolitan documents are made from sections. Each one
start with two dashes followed by a name. For example, 
here's a title section:







-- title



A Basic Example



This is an example of what a Neopolitan

document looks like.






which outputs this HTML:






<h1 class="title">

  A Basic Example

</h1>

<p>

  This is an example of what a

  Neopolitan document looks like.

</p>

Section Flags

Each section can have flags. They're defined by adding lines starting with two dashes immediately follow the section like this:





-- h2

-- hidden 





This H2 is hidden








Which outputs:








<h2 hidden>

  This H2 is hidden

</h2>

Section Attributes

Key/value attributes can be added as well. For example:





-- h2

-- class: red 





This H2 is red 








Which produces:








<h2 class="red">

  This H2 is red

</h2>

Markdown Section Comparison

A basic Markdown document might look like this:





# The Title





Some text.





## Forward





More text.








A corresponding Neopolitan document 
would look like this:








-- h1





The Title





Some text.





-- h2




Forward





More text.








Each would produce:








<h1>The Title</h1>

<p>Some Text.</p>

<h2>Forward</h2>

<p>More Text.</p>








Neopolitan takes more lines to produce 
the same output. It provides no benefit if 
that's all you need.

Enter Neopolitan

Neopolitan's usefulness comes from adding HTML features to content1.

For example, changing the above demo to use explicit sections and attributes like this:





-- title





The Title





Some text.





-- forward

-- class: blue






More text.








produces this:








<h1 class="title" aria-label="Page Title">

  The Title

</h1>

<p>Some Text.</p>

<h2 class="forward blue">

  Forward

</h2>

<p>More Text.</p>








Specifically, it does the following:




    
  
  • 
    Automatically add the "title" class 
    to the "h1" element
  
    • 
  
  • 
    Automatically add an "aria-label" to
    the "h1" element2
  
    • 
  
  • 
    Output an H2 with the "Forward" text in
    it without having to write the word in addition 
    to using it as the section name3.
  
    • 
  
  • 
    Automatically add the "forward" class
    to the H2 
  
    • 
  
  • 
    Append the "blue" class to the H2 
    via the "-- class: blue"
    section attribute
  
    • 



Of course, the same output could be produced with
Markdown. It would require falling back to
raw HTML for most of it, though.

Tags

The Basics




Neopolitan tags provide a way to enhance content. 
A basic one looks like this:





  <<tag-name|some text>>





  The tag-name defines what the type
  of tag. The some text in this example
  represents the content the tag works on.




Element Examples



Basic examples, include:





  <<strong|some text>>

  <<em|some text>>

  <<kbd|some text>>





Which output:




  some text

  some text

  some text

Custom Tags




  Those all map to existing HTML elements. 
  They don't have to. Tags can be used to do
  anything with the content4. They
  can also include content generated
  by the output system itself. For example,
  a wiki tag could be set
  up like this:





  <<wiki|Documentation>>





The app generating the content could automatically 
transform that into a link to the Documentation page, 
add some styles, an id, and a data attribute.
Something like:





  
  <a

  href="/documentation.html"

  id="documentation-link"

  class="wiki-link"

  data-group="examples"

  >Documentation<<a>





The content stays readable while the
HTML has all the parts necessary
to provide the functionality




Attributes and Flags




  Tags can use attributes and flags. 
  They're made by adding a | at
  the end of the content to separate them. 
  For example, a link tag needs a 
  URL. That's done by adding it as a flag like this:





  <<link|Example Site|https://www.example.com/>>




(aside: It may feel weird to call a URL
a flag if you've done a lot of coding.
In terms of Neopolitan, "flag"
just means a string of content instead
of a key value pair. You can 
learn more about them on 
Attributes and Flags page.)






Attributes work the same way. The
only difference is they are key/value
pairs. For example, this is how to use
a tag attribute to add a CSS class to the output:





  <<kbd|Escape|class: red>>





Which produces: 





  <kbd class="red">Escape<kbd>

Multiple Attributes And Flags




  Any number of attributes and flags can be added to 
  a given tag5. All that's needed is
  another | separator between them.
  For example:





  <<link

  |example text

  |https://www.example.com/

  |class: red

  |id: alfa

  |hidden>>




Which produces:




  <a

  href="https://www.example.com/"

  class="red"

  id="alfa"

  hidden

  >example text</a>

Shorthands

The Basics




  Shorthands provide another way to enhance
  content in Neopolitan files. Each works
  like a specific Tag, but without the 
  need to type the full thing. For example, 
  one shorthand uses **.
  It looks like this:




**example text**

Which outputs this HTML:

<strong>example text</strong>

Which is the exact same thing you'd get if you used a tag like this:

<<strong|example text>>

Attributes and Flags

Shorthands can use attributes and flags the same way Tags can. For example the `` shorthand is for code snippets. Adding a flag to it with the name of a language adds the appropriate syntax highlighting:

``println!("ping");|rust``

Adding an attribute to a strong shorthand looks like this:

``example text|class: red``

Which outputs:

<strong class="red">example text</strong>

Multiple Attributes and Flags

Shorthands can use multiple attributes and flags as well. For example, the link shorthand uses [[ and ]]. It uses a flag by default for the URL. Styles and IDs can be added like this:

[[example link
|https://www.example.com/
|id: example-link
|class: red]]

Which outputs:

<a
href="https://www.example.com/"
id="example-link"
class="red"
>example link</a>

The Shorthand List

Neopolitan provides nine shorthands. They are:

  • Span:

    @@code snippet@@

  • Code:

    ``code snippet``

  • Strong:

    **strong text**

  • Emphasis:

    __emphasis text__

  • Footnote Reference:

    ^^1^^

    See: the footnote documentation for more details on how footnotes work

  • Deleted:

    ~~deleted text~~

  • Inserted:

    ((inserted text))

  • Link:

    [[link text|https://www.example.com/]]

  • Image:

    {{image-name.jpg|alt: Alt text for the image}}
