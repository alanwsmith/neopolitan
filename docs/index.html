Welcome to Neopolitan

A plain-text format for making web sites.

It's like Markdown, but with more features.

How It Works

The Basics

Neopolitan documents are made from sections. Each one start with two dashes followed by a name. For example:

-- title

A Basic Example

This is an example of what a Neopolitan
document looks like.

The HTML output looks like this:

<h1 class="title">
  A Basic Example
</h1>
<p>
  This is an example of what a
  Neopolitan document looks like.
</p>

Section Flags

Each section can have flags. They're defined by adding lines starting with two dashes immediately follow the section like this:

-- h2
-- hidden

This H2 is hidden

Which outputs:

<h2 hidden>
  This H2 is hidden
</h2>

Section Attributes

Key/value attributes can be added as well. For example:

-- h2
-- class: red

This H2 is red

Which produces:

<h2 class="red">
  This H2 is red
</h2>

Markdown Section Comparison

A basic Markdown document might look like this:

# The Title

Some text.

## Forward

More text.

A corresponding Neopolitan document would look like this:

-- h1

The Title

Some text.

-- h2

Forward

More text.

Each would produce:

<h1>The Title</h1>
<p>Some Text.</p>
<h2>Forward</h2>
<p>More Text.</p>

Neopolitan takes more lines to produce the same output. It provides no benefit if that's all you need.

Neopolitan Benefits

Neopolitan's usefulness comes from adding HTML features to content1.

For example, changing the above demo to use explicit sections and attributes like this:

-- title

The @@Title|class: red@@

Some text.

-- forward
-- class: blue

More text.

produces this:

<h1 class="title" aria-label="Page Title">
  The <span class="red">Title</span>
</h1>
<p>Some Text.</p>
<h2 class="forward blue">
  Forward
</h2>
<p>More Text.</p>

Specifically, this output does the following:

  • Automatically add the "title" class to the "h1" element
  • Automatically add an "aria-label" to the "h1" element2
  • Use a span to add a "red" class to the word Title in the H1. This is done with the "@@" shorthand which is described further below. The key point for this example is the ability to add styles inside headlines.
  • Output an H2 with the "Forward" text in it without having to write the word in addition to using it as the section name3.
  • Automatically add the "forward" class to the H2
  • Append the "blue" class to the H2 via the "-- class: blue" section attribute

Of course, the same output could be produced with Markdown. But, it would require falling back to raw HTML for most of it.

Footnotes

  1. Sure, you could always roll back to using HTML directly in your Markdown documents.
  2. Adding the "aria-label" is a bit of a contrived example. I don't know if including one in an H1 is advisable. The point here is not about the specific label. It's about the general ability to add things like aria labels during output without having to define them directly in the content itself.

  3. The names don't have to match. You could just as easily make a section called "-- week" and have the output be "The Week In Review".

    There's also no restriction limiting the output to an H2 tag. An entire structure of divs could be used the same way.

Section Attribute Example

Neopolitan documents consist of sections. They start with two dashes followed by a name (e.g. -- title, -- h2, -- image, etc.). Each section can have key/value attributes and flags. They also start with two dashes and look like ``-- attrKey: value`` and ``-- flagName``.

For example, this section has a ``-- class: red`` attribute. As a result, the HTML output looks like:

-- code
-- html

<h2 class="red">
  Section Attribute Example
</h2>

-- footnote
-- id: 1

This is an example footnote. They are linked to the content in the document by matching the -- id attribute of the section to the corresponding ^^ID^^ shorthand in the document.
