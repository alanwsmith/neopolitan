Introduction

The specification for Neopolitan 2.0.0 is test based. Any parser that passes all the test meets the specification.

Test Types

The specification is split into three parts.

1. AST Generation

2. Required Sections and Attributes

3. HTML Output

AST Generation

Tests are currently built into the rust code. They include parsing for the shorthands.

Sections can have attrs (attributes) and flags. The flags must preserve the order from the source input. Attrs are key/value pairs with no requirement for maintaining a specific order.

Neopolitan is designed primarily to generate web content. Output is largely an implementation detail with a required exceptions designed to improve portability between systems.
