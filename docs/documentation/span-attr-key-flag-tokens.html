Span Attr Key/Flag Tokens

Tags and Shorthand spans accept flags and attributes. The parser uses the same token logig for individual flags and the key portion of attributes.

The specific logic is:

- The token ends at spaces, tabs, newlines or pipes [TKTKTK: Insert pipe character]

- Escaping character in tokens is not supported. Doing so would require switching the AST to use an array of spans to track the escaped characters instead of using a single text string. That would lead to an unacceptable increase in the complexity of using them in templates.
