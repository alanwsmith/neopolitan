attributes section

The -> attributes section does not output directly on the page. It contains metadata in the form of key/value pairs that can be used as output material or for processing purposes.

I use it for the ID, date, status, etc... The permitted and parsed values are left as an implementation detail. My implementation is straight attributes like the rest of the sections but YAML or other formats could be used with a corresponding implementation engine.

Example

-> attributes
>> date: 2023-04-15 03:28:24
>> id: 2osaexpae3om
>> site: neopolitan 
>> type: example 
>> status: published
