Introduction

My site is twenty years old. Pondering those decades got me thinking about the next two. I find myself asking "What do I want to use for the next twenty years?" instead of "What do I want to do for my next redesign?" The answers are very different.

The Format

I've moved my site a half dozen times. Each one involved a painful content migration. I've gone from raw HTML to Perl scripts to Wordpress to PHP with various DB schemas to Markdown to MDX to Org-Mode to a mix of all of the above.

I want aspects of each of those rolled into a single format. I haven't found anything like that. So, I'm rolling my own. It's called Neopolitan and it looks like this:

-> title

Welcome To Neopolitan

-> subtitle

A plain-text format for websites

-> note
>> class: highlighted

Content Sections

Neopolitan documents are divided into sections. Each one starts with a -> token followed by the name of the section. The type of section determines what happens with the content inside it. For example, the -> h2 section becomes an <h2> element:

Source
-> h2

Bring your best compass
Output
<h2>Bring your best compass</h2>

While -> list sections use - characters to define list items:

Source
-> list

- Raise the sail

- Shut the hatch
Output
<ul>
  <li>
    <p>Raise the sail</p>
  </li>
  <li>
    <p>Shut the hatch</p>
  </li>
</ul>

HTML Equivalents

Neopolitan provides equivalent sections for most basic HTML tags. For example, -> aside which works like this:

Source
-> aside

Tear a thin sheet from the yellow pad.
Output
<aside>
  <p>Tear a thin sheet from the yellow pad.</p>
</aside>

Raw HTML

An -> html section provides the ability to pass raw HTML directly to the final output

Source
-> html

<form>
    <label for="name">Enter your name: </label>
    <input type="text" name="name" id="name" required>
</form>
Output
<form>
    <label for="name">Enter your name: </label>
    <input type="text" name="name" id="name" required>
</form>

Meta Content

Section with differen types of metadata are included. -> attributes sections provide key/value pairs at the page level to the processing engine. Beyond that, -> css and -> script sections push their content into the head of the document so pages can be adjusted individually.

Source
-> css

.highlight {
  background: white;
}
Output
<style>
.highlight {
  background: white;
}
</style>

Inline Tags

Several inline tags are provided for styling, linking, etc...

Source
-> p

The <<strong|colt>> reared.
Output
<p>The <strong>colt</strong> reared.</p>

An AST Not An Output

The HTML outputs shown above are from my processing implementation. They're an optional starting point, not a spec. Neopolitan output is an implementation details. The thing that has the spec is the AST that's generated by parsing a document.

Section Types

These are the available section types

The checkboxes are for me to know which ones still need samples

  • aside - The HTML <aside> tag

  • attributes - Metadata for the page

  • blockquote - The HTML <blockquote> tag

  • blurb - Not rendered on the page. Used as meta data for links into the page.

  • code - Code blocks like fenced code blocks but you can add an arbitrary attributes to them.

  • comment - Parsed like text sections in the AST but with the expectation that they won't output unless the implementation is specifically designed to do so.

  • css - A place to put CSS that will be inlined in the page

  • div - A <div> wrapper that can be used to add attributes to sections of content

  • h1-h6 - HTML heading tags

  • html - for raw html blocks that go directly to the output without further processing

  • image - An image call with the alt text coming from the body of the section

  • list - Unordered lists

  • neoexample - A display for examples on the neopolitan documentation site

  • note - A basic div wrapper with a note attribute and any other attributes that are passed in

  • notes - A list style notes section. Same as an unordered lists but with a div around it in the output (based on implementation)

  • olist - An ordered list

  • p - Basic text paragraph tags that are not otherwise contained in a div.

  • pre - A pre tag

  • results - Used to show output from -> code blocks

  • subtitle

  • startcode/endcode

  • textarea

  • title

  • todo - TODO lists

  • vimeo - A vimeo embed

  • warning - A div block with a warning class on it

  • youtube - A youtube embed
