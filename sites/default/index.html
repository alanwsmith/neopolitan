Introduction

The Format

I've moved my site a half dozen times. Each one involved a painful content migration. I've gone from raw HTML to Perl scripts to Wordpress to PHP with various DB schemas to Markdown to MDX to Org-Mode to a mix of all of the above.

I want aspects of each of those rolled into a single format. I haven't found anything like that. So, I'm rolling my own. It's called Neopolitan and it looks like this:

-> title

Welcome To Neopolitan

-> subtitle

A plain-text format for websites

-> note
>> class: highlighted

Content Sections

Neopolitan documents are divided into sections. Each one starts with a -> token followed by the name of the section. The type of section determines what happens with the content inside it. For example, the -> h2 section becomes an <h2> element:

Source
-> h2

Bring your best compass
Output
<h2>Bring your best compass</h2>

While -> list sections use - characters to define list items:

Source
-> list

- Raise the sail

- Shut the hatch
Output
<ul>
  <li>
    <p>Raise the sail</p>
  </li>
  <li>
    <p>Shut the hatch</p>
  </li>
</ul>

HTML Equivalents

Neopolitan provides equivalent sections for most basic HTML tags. For example, -> aside which works like this:

Source
-> aside

Tear a thin sheet from the yellow pad.
Output
<aside>
  <p>Tear a thin sheet from the yellow pad.</p>
</aside>

Raw HTML

An -> html section provides the ability to pass raw HTML directly to the final output

Source
-> html

<form>
    <label for="name">Enter your name: </label>
    <input type="text" name="name" id="name" required>
</form>
Output
<form>
    <label for="name">Enter your name: </label>
    <input type="text" name="name" id="name" required>
</form>

Meta Content

Section with differen types of metadata are included. -> attributes sections provide key/value pairs at the page level to the processing engine. Beyond that, -> css and -> script sections push their content into the head of the document so pages can be adjusted individually.

Source
-> css

.highlight {
  background: white;
}
Output
<style>
.highlight {
  background: white;
}
</style>

Inline Tags

Several inline tags are provided for styling, linking, etc...

Source
-> p

The <<colt|strong>> reared.
Output
<p>The <strong>colt</strong> reared.</p>

An AST Not An Output

The HTML outputs shown above are from my processing implementation. They're an optional starting point, not a spec. Neopolitan output is an implementation details. The thing that has the spec is the AST that's generated by parsing a document.
