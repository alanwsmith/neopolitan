Inline Tags And Shorthands

Inline Tags

Several inline tags are provided for styling, linking, etc...

Source
-> p

The <<colt|strong>> reared.
Output
<p>The <strong>colt</strong> reared.</p>

Inline Tag Attributes

Key/value pairs passed to inline tags output as attributes:

Source
-> p

The <<tree top|strike|id: alfa|class: bravo>> waved
Output
<p>The <strike id="alfa" class="bravo">tree top</strong> waved</p>

Inline Shorthand

Shorthand versions of code, em, strong, and link look like this:

Source
-> p

`alfa``

_bravo__

*delta**

>charlie>https://www.example.com/>
Output
<p><code>alfa</code></p>
<p><em>bravo</em></p>
<p><strong>delta</strong></p>
<p><a href="https://www.example.com/">charlie</a></p>

Inline Shorthand Attributes

Attributes can also be added to the shorthand tags:

Source
-> p

`echo`id: foxtrot|class: highlighted`
Output
<p><code id="foxtrot" class="highlighted">echo</code></p>

Inline Attributes

Adding attributes to inline tags is also possible:

Source
-> p

alfa `bravo`class: charlie` delta
Output
<p>alfa <code class="charlie">bravo</code> delta</p>

Inline Code Language Shorthand

If a key is passed as the first argument to a code shorthand without a value it's treated as a language and turned into a language-x class in the HTML.

Source
-> p

`delta`rust`

`echo`rust|id: sierra`
Output
<p><code class="language-rust">delta</code></p>
<p><code class="language-rust" id="sierra">echo</code></p>

NOTE: Adding an explicit class along with the language currently results in two class attributes being added to the output. That is, this would be a problem:

Source
-> p

`echo`rust|class: highlighted`
Output
<p><code class="language-rust" class="highlighted">echo</code></p>
