Neopolitan Sections

These are the available section types (things that aren't linked aren't yet implemented)

  • aside The HTML <aside> tag

  • attributes - Metadata for the page

  • blockquote - The HTML <blockquote> tag

  • blurb - Passed to other pages for incoming links

  • canvas - Basic canvas element with included attributes

  • categories - A metadata section for page categories

  • checklist - Like a TODO list, but it resets every time the page is visited

  • code - Code blocks like fenced code blocks but you can add an arbitrary attributes to them.

  • comment - Section that's not output

  • css - A place to put CSS that will be inlined in the page

  • details - The details disclosure element: <details>

  • div - A <div> wrapper that can be used to add attributes to sections of content

  • dlist - The HTML <dl> tag

  • ext - Call to an external process

  • figure - The HTML figure element

  • h1-h6 - HTML heading tags

  • head - raw content to put in the head of the document

  • html - for raw html blocks that go directly to the output without further processing

  • image - An image call with the alt text coming from the body of the section

  • list - Unordered lists

  • menu - the HTML <menu> tag

  • nav - the HTML <nav> tag

  • neoexample - A display for examples on the neopolitan documentation site

  • note - A div wrapper with a class="note" attribute

  • notes - A list style notes section

  • object - The HTML <objet> tag

  • olist - An ordered list

  • p - Basic text paragraph tags that are not otherwise contained in a div.

  • picture - The HTML <picture> element

  • pre - A pre tag

  • reference - For documenting external resources

  • results - Used to show output from -> code blocks

  • startcode/endcode

  • script - A raw section that adds script tags to the header of the page

  • subtitle

  • table - Tables - textarea

  • title

  • todo - TODO lists

  • video - The HTML <video> element

  • vimeo - A vimeo embed

  • warning - A div block with a warning class on it

  • widget - An area to pass data to the client or build processes for custom functionality

  • youtube - A youtube embed
